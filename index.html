Cours GIT

Qu'est-ce que GIT

GIT est un outils de versionning et il permet d'avoir une tracabilité et un historique tous les fichiers du projet

Il permet également de travailler en équipe: Envoyer nos modifications a nos collègue et récupérer les modifications des collegues.

Dans la tracabilité des fichiers GIT nous donne toujours 3 infos : Qui, Quand, Pourquoi ?

Les Commandes de base

Initialiser un projet:
git init
Voir le statut des fichiers
git status
Ajouter des fichiers
git add 'fichier'
Ajouter TOUS LES FICHIERS :
git add .
Commit (enregistrer) les fichier ajouter
git commit
Un commit nécéssite toujours un message
commit avec message:
git commit -m 'nom du fichier'
commit avec add + message (fonctionne sur les fichiers déja add précédemment):
git commit -a -m 'nom du fichier'

Correction des fichiers

Voir les différents commit :
git log
Corriger les modifications sur un fichier (pas encore add) :
git checkout 'fichier'
Supprime un add :
git reset 'fichier'
git reset .
Supprimer le dernier commit :
git reset HEAD^
Supprimer les deux dernier commit :
git reset HEAD^^

La notion de branche

Une branche est une copie du projet, Elle permet de développer en parralèlle de nouvelles fonctionnalités. Lorsqu'on considère les nouveautés opérationnelle, on peut les rappatrier sur la version principale (master)

Voir les branches:
git branch
Créer une nouvelle branche :
git branch 'nom'
Se placer sur une branche :
git checkout 'nom'
Voir les différences entre 2 branches:
git diff 'nom'..master
Apporter les modif effectués sur une branche dans master (il faut etre sur master pour effectuer cette commande)
git checkout master
git fetch 'nom'
Le fetch va littéralement remplacer master par la branche. Il s'agit donc d'une grosse modification dans le projet (ex: version 2)
